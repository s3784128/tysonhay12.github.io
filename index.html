Project Idea

Overview
The project I wish to make is a smartphone rostering app that could be used to make the rostering system at a business better than just a paper system. This would allow employees to easily change shifts if they are unavailable to work, while also giving other employees a fair chance to pick up the shift, as this would notify them through the app. This would also show the full schedule for each week, which would help employees remember when they are working.

Motivation
My motivation behind this idea is that my current workplace, Templestowe IGA, the rostering system is inefficient and slightly unfair as casual workers like myself can rarely pick up extra shifts as the shifts are usually already taken by the part time employees because the system is dependant on whoever luckily sees it first in the office. I do not expect that this software would be used, but if I manage to develop it to a high standard, there is a chance it may be adopted by my employers. This would also get a positive response from the other employees who also go to university, as we have agreed that the current system does not allow us to obtain the number of shifts that we should be getting (life’s definitely my expensive after finishing high school).

Description
The application would include options to swap shifts, get shifts covered, give a whole weekly schedule and giving notifications to the devices. It would have a simple interface with large text/buttons so that is easily usable by all employees, regardless how tech savvy they are. The notifications would be quite integral to the application as it would be the first way to notify employees when they are able to take new shifts. There should be the option to apply for long holiday periods, making it easily for the managers to know when employees will be unavailable in the long term. It would be developed so it could be used on both iOS and Android so that regardless of the device used, every employee could use it. The application shouldn’t be too flashy and colourful as I believe not all employees will wish to use it if its too complicated.

Tools and Technology
From some research, it appears that I will need to install how to use app development software, such as one software I found called Firebase which helps with development of real-time scheduling applications. I will use my laptop and RMIT desktops to develop my application.

Skills Required
The skills required would be a much better understanding on HTML coding, while also needing to learn coding in AngularJS or CSS (languages similar or based on HTML).

Outcome
If this project is successful, it would result in a more effective and efficient rostering system for my workplace, therefore leading to better employee satisfaction. This would solve the issue of casual workers not getting enough shifts and allowing employees to request for a shift to be covered/swapped without needing to go into the workplace. Although, I expect for my employers to allow this software to be used, it would need to look professional and easily usable for also the older/less tech savvy employees (not trying to sound rude, but the paper system is preferred by the older employees as I believe they do not want to adopt new technology).
